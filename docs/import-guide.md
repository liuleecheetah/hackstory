# HackStory 匯入格式說明

這份說明寫給要把自己的資料做成時間軸的人。你不需要懂程式——只要會用試算表（Google Sheet 或 Excel）就夠了。

---

## 一、三種把資料放進 HackStory 的方式

| 方式 | 適合的情況 | 在哪裡操作 |
|---|---|---|
| 上傳 CSV 檔案 | 資料在 Excel 或電腦裡 | 上方「匯入 CSV / Google Sheet」按鈕 |
| 貼上 Google 試算表網址 | 資料在 Google Sheet、想多人共同編輯 | 同上 |
| 載入 `.hst.json` 檔案 | 別人給你的 HackStory 時間軸檔 | 左側圖層面板「＋載入 .hst.json」 |

前兩種都會先進入「**匯入預覽**」畫面，讓你看到哪些資料成功、哪些有問題，確認後才變成圖層。

---

## 二、試算表要長什麼樣子

**第一列是表頭，之後每一列是一則事件。** 只有兩個欄位是必要的：**日期**和**標題**，其他都可以省略。

### 欄位對照表

表頭中英文都認得，大小寫不拘：

| 欄位 | 接受的表頭寫法 | 必填 | 說明 |
|---|---|---|---|
| 日期 | `日期`、`開始日期`、`Date`、`Start Date` | ✅ | 事件發生的時間（寫法見下一節） |
| 標題 | `標題`、`事件`、`Title` | ✅ | 一句話講完。這是軸上會直接顯示的文字 |
| 時間 | `時間`、`Time`、`Start Time` | | 幾點幾分，例如 `13:00`；也可寫時段 `09:00-18:00` |
| 結束日期 | `結束日期`、`End Date` | | 填了就變成「一段期間」，畫成長條而不是圓點 |
| 說明 | `說明`、`描述`、`Description` | | 完整的內容，會保存在檔案裡 |
| 地點 | `地點`、`Location` | | 例如 `立法院前` |
| 分類 | `分類`、`軸線`、`Track` | | 相同分類的事件會排在同一條軸線上（見第五節） |
| 標籤 | `標籤`、`Tags` | | 用逗號分隔，例如 `釋憲, 大法官` |
| 來源 | `來源`、`Source`、`URL` | | 資料出處。貼網址最好，也可以寫文字 |

認不得的表頭會被忽略（該欄資料不會匯入），匯入預覽畫面會告訴你哪些欄位有被認出來。

### 範例

| 日期 | 時間 | 結束日期 | 標題 | 分類 | 來源 |
|---|---|---|---|---|---|
| 1986/3/7 | | | 祁家威請求與男性公證結婚遭拒 | 立法 | |
| 2006/10/11 | | 2006/10/31 | 蕭美琴提《同性婚姻法》草案遭退回 | 立法 | |
| 2016/11/24 | 09:00-18:00 | | 司法及法制委員會召開公聽會 | 立法 | |
| 2016/12/10 | 13:00 | | 婚姻平權音樂會 | 運動 | |
| 2017/5/24 | | | 釋字第 748 號公布 | 立法 | https://cons.judicial.gov.tw/... |

## 三、日期怎麼寫

### 都可以的寫法

分隔符號用斜線、橫線、點、國字都行，補零不補零都行：

| 你寫的 | HackStory 的理解 |
|---|---|
| `2017/5/24`、`2017-05-24`、`2017.5.24`、`2017年5月24日` | 2017 年 5 月 24 日 |
| `2010年6月`、`2010/6`、`2010-06` | 2010 年 6 月（精確到「月」） |
| `1986`、`1986年` | 1986 年（精確到「年」） |

### 重要觀念：不知道確切日期，就不要假裝知道

真實世界的資料常常只知道「大概什麼時候」。**只知道年月就只寫年月**，HackStory 會誠實地把它記成「精確到月」，而不是隨便塞一個日期。這是刻意設計，不是限制。

### 時間欄

- 單一時刻：`13:00` → 事件標在當天 13:00
- 時段：`09:00-18:00` → 自動變成「當天 09:00 到 18:00」的區間事件
- 日期和時間寫在同一格也可以：`2016/12/10 13:00`

### 不合法的例子（會被攔下來，不會混進時間軸）

- `2017/2/30`（2 月沒有 30 日）、`2017/13/1`（沒有 13 月）、`25:00`
- `你好天`、`大概是夏天` 這類文字——會進「待修正」清單讓你改（見下一節）

## 四、遇到不完整的資料會怎樣

**HackStory 的原則：絕不偷偷丟掉你的資料。** 每一列的下場都會攤開給你看：

| 你的資料長這樣 | HackStory 的處理 |
|---|---|
| 整列空白 | 自動略過，不吵你 |
| 有日期、沒標題 | 略過，但記在「警告」清單提醒你 |
| 日期看不懂（如 `你好天`）| 進紅色「**待修正**」區：欄位可以**直接在預覽畫面改**，按「重試」就好；不想要就按「忽略」 |
| 欄位整列錯位 | 同上，進「待修正」 |
| 同一天、同標題出現好幾列 | **全部保留**，標黃色「疑似重複」，由你決定要不要按 ✕ 移除——程式不會自作主張刪資料 |

匯入按鈕會誠實寫明狀況，例如「匯入 27 筆為新圖層（放棄未修正 2 筆）」。

---

## 五、分類欄：一份表格畫成好幾條軸線

如果你的表格有「分類」欄，**相同分類的事件會排在同一條軸線**，例如「立法」一條、「運動」一條，上下並排對比。

沒有分類欄的話，全部事件會放進同一條叫「匯入資料」的軸線。

---

## 六、用 Google 試算表要先做一件事：設成公開

HackStory 不會要求你登入 Google（也不碰你的帳號權限），所以試算表必須是「公開可讀」的。兩種設法擇一：

**方法 A（推薦）：發布到網路**
1. 在 Google Sheet 裡：檔案 → 共用 → **發布到網路**
2. 按「發布」，複製產生的連結

**方法 B：連結共用**
1. 右上「共用」→ 一般存取權改成「**知道連結的使用者**」（檢視者即可）
2. 複製瀏覽器網址列的網址

把連結貼進 HackStory 的匯入視窗按「讀取」即可。

**注意：一次匯入一個分頁（工作表）。** 想匯入特定分頁，先在瀏覽器切換到那個分頁再複製網址。多個分頁就匯入多次，每個分頁會變成一個圖層——正好拿來疊加對比。

## 七、`.hst.json` 檔案（進階）

這是 HackStory 自己的時間軸檔案格式：一條時間軸就是一個獨立檔案，可以下載、備份、寄給別人、疊加混合。

- 從試算表匯入後，用「匯出／分享」就能下載成 `.hst.json`
- 拿到別人的 `.hst.json`，用左側面板「＋載入 .hst.json」疊上去對比
- 檔案格式的完整技術規格在專案的 [SPEC.md](../SPEC.md)（給想手寫或寫程式產生檔案的人）

**關於授權：** 匯入的資料預設會標記 CC-BY-4.0 授權。若你的資料另有授權條件，請留意這點——時間軸檔案是設計來被分享與再利用的。

## 八、常見問題

**問：說明、地點、標籤匯入後去哪了？**
答：**點一下時間軸上的事件**就會浮出詳情卡，完整顯示說明、地點、標籤、資料來源與查證程度。這些欄位也都完整保存在檔案裡。

**問：匯入後發現打錯字怎麼辦？**
答：目前請改試算表原稿再重新匯入一次（舊圖層按 ✕ 移除）。

**問：可以匯入幾筆資料？**
答：幾百筆沒問題。太密的事件在縮小時會自動往下疊排，放大就看清楚了。
